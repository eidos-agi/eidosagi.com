## Claim

Improving data quality is more effective than improving models. Andrew Ng defines data-centric AI as 'the discipline of systematically engineering the data needed to build a successful AI system.' His research shows high-quality smaller datasets outperform large messy ones. A steel defect detection study found model-centric approach failed to improve accuracy while data-centric approach boosted it by 16%. Ng argues 'all that progress in algorithms means it's actually time to spend more time on the data.' This directly supports MVD: the data specification is where value concentrates.

## Disconfirmation Search

For LLM pretraining at massive scale, model architecture still matters. But for domain-specific tasks with structured outputs (grocery ordering, code review, deployment), the data-centric finding holds strongly. MVD targets these structured-output domains.
