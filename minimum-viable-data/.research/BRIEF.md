## Key Findings

- **Sutton's bitter lesson: hand-crafted knowledge always loses to general methods** — General methods that leverage computation consistently outperform hand-crafted human knowledge as compute scales. Demonstrated in chess (Deep Blue's simple alpha-beta search beat expert-encoded systems), Go (AlphaGo Zero eliminated human expertise entirely, trained by self-play alone), and speech (general hidden Markov models beat 1970s hand-crafted approaches). Skills files are hand-crafted knowledge for AI. Contracts are the general method — they specify the goal and let the model's increasing capability determine the approach.
- **Data-centric AI outperforms model-centric AI** — Improving data quality is more effective than improving models. Andrew Ng defines data-centric AI as 'the discipline of systematically engineering the data needed to build a successful AI system.' His research shows high-quality smaller datasets outperform large messy ones. A steel defect detection study found model-centric approach failed to improve accuracy while data-centric approach boosted it by 16%. Ng argues 'all that progress in algorithms means it's actually time to spend more time on the data.' This directly supports MVD: the data specification is where value concentrates.
- **AI-generated code is a new category of liability — code volume outpaces audit capacity** — Code is becoming a liability at unprecedented scale. AI writes 30% of Microsoft's code and 25%+ of Google's. Anthropic's 2026 report shows agents working for days building entire applications, enabling 'full application builds, backlog cleanup, and technical debt reduction' — but also creating code faster than humans can audit. Agents now review AI-generated code 'for security issues, consistency, and defects at a scale humans cannot match.' MIT Technology Review named generative coding a 2026 Breakthrough Technology. The industry response to too much code is more code (agents reviewing agents). MVD breaks this cycle by eliminating the code layer where structured data suffices.
